Data Engineering Fundamentals

15 questions
Q1
What is the difference between ETL and ELT? When would you choose one over the other?
Answer

ELT is preferred when you have a powerful data warehouse (Synapse, BigQuery, Snowflake) that can handle transformations at scale. ETL is used when you need to transform data before loading to save storage costs or when the destination has limited compute.

Q2
Explain the Medallion Architecture (Bronze/Silver/Gold). Why do companies use it?
Answer

Bronze = raw data (never modified), Silver = cleaned/validated, Gold = aggregated/business-ready. It provides clear data lineage, allows reprocessing from raw, and makes debugging much easier.

Q3
What is a data lake vs. a data warehouse? What is a lakehouse?
Answer

Lake = cheap, flexible, stores everything raw. Warehouse = structured, fast, expensive. Lakehouse = Delta Lake/Iceberg adds ACID transactions on top of a data lake, combining both.

Q4
What is the difference between batch processing and stream processing?
Answer

Batch = scheduled runs on accumulated data. Stream = continuous processing of events as they arrive. Different tools, different latency requirements.

Q5
How would you handle late-arriving data in a streaming pipeline?
Answer

Watermarking, event time vs. processing time, allowing a late data window. In Spark Structured Streaming, use watermarks to bound state.

Azure-Specific Questions

12 questions
Q1
Explain the difference between Azure Data Factory and Azure Databricks. When do you use each?
Answer

ADF = orchestration and data movement (no-code/low-code). Databricks = heavy data transformation with PySpark. In practice, ADF orchestrates Databricks notebooks.

Q2
What is Delta Lake and why do you use it over plain Parquet?
Answer

Delta Lake adds ACID transactions, schema enforcement, time travel, and efficient upserts (MERGE) to Parquet files. It makes data lakes reliable.

Q3
How do you handle secrets and credentials in an ADF + Databricks pipeline?
Answer

Azure Key Vault linked service in ADF, Key Vault secret scope in Databricks. Never hardcode anything. Use managed identities between Azure services.

Q4
What is the difference between ADLS Gen2 and Azure Blob Storage?
Answer

ADLS Gen2 is built on Blob Storage but adds a hierarchical namespace (real folders), ACLs for fine-grained access control, and is optimized for analytics workloads.

SQL Interview Questions

10 questions
Q1
Write a query to find the second highest salary in an employees table.
Answer

Use DENSE_RANK() or subquery: SELECT MAX(salary) FROM employees WHERE salary < (SELECT MAX(salary) FROM employees). DENSE_RANK is cleaner.

Q2
What is the difference between RANK() and DENSE_RANK()?
Answer

RANK() leaves gaps after ties (1,1,3). DENSE_RANK() does not (1,1,2). Most interview questions want DENSE_RANK unless specifically asking for RANK.

Q3
Explain the difference between WHERE and HAVING.
Answer

WHERE filters rows before aggregation. HAVING filters groups after aggregation. You cannot use aggregate functions (SUM, COUNT) in a WHERE clause.

System Design

8 questions
Q1
Design a data pipeline that processes 1 million sales transactions per day. What would your architecture look like?
Answer

Source → ADF (ingest, schedule) → ADLS Gen2 Bronze → Databricks (transform, Delta) → Silver/Gold → Synapse Analytics → Power BI. Include Key Vault, monitoring, retry logic.

Q2
How would you design a system that needs both real-time and historical analytics?
Answer

Lambda Architecture: batch layer (ADF + Databricks) for historical, speed layer (Event Hubs + Stream Analytics) for real-time. Serving layer (Synapse) merges both.
