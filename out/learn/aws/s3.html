Amazon S3 — AWS Data Lake Storage

Amazon S3 is the foundation of every AWS data engineering architecture. It is the data lake where Bronze, Silver, and Gold layers live. Simple, infinitely scalable, and cheap — every AWS pipeline starts and ends here.

S3 vs. ADLS Gen2 — key differences

Amazon S3 (Simple Storage Service) is AWS's object storage service — the direct equivalent of Azure Data Lake Storage Gen2. Both store files at any scale, both are cheap, both integrate natively with all the compute services in their respective clouds.

The main structural difference is that S3 uses buckets where ADLS Gen2 uses containers. In S3, you typically create separate buckets per environment or per layer (bronze-bucket, silver-bucket, gold-bucket). In ADLS Gen2, you typically use one storage account with multiple containers. Both approaches work — it's an organizational preference.

🥉
Bronze Bucket

Raw data exactly as received from source systems. One bucket, organized by source/date. Never deleted, never modified.

🥈
Silver Bucket

Cleaned and validated data. Stored as Parquet or Delta Lake files. Partitioned by date for query efficiency.

🥇
Gold Bucket

Aggregated business-ready data. Queried by Athena and Redshift. Optimized for read performance.

Setting up S3 for a data lake

create_s3_buckets.py
python
# Boto3 — Python SDK for AWS
# Create S3 buckets for Medallion Architecture

import boto3

s3 = boto3.client('s3', region_name='us-east-1')

# Create three buckets — one per Medallion layer
for layer in ['bronze', 'silver', 'gold']:
    s3.create_bucket(Bucket=f'yourcompany-datalake-{layer}')
    
    # Block all public access (mandatory in production)
    s3.put_public_access_block(
        Bucket=f'yourcompany-datalake-{layer}',
        PublicAccessBlockConfiguration={
            'BlockPublicAcls': True,
            'IgnorePublicAcls': True,
            'BlockPublicPolicy': True,
            'RestrictPublicBuckets': True
        }
    )
    
    # Enable versioning for auditability
    s3.put_bucket_versioning(
        Bucket=f'yourcompany-datalake-{layer}',
        VersioningConfiguration={'Status': 'Enabled'}
    )

print("✅ Bronze, Silver, Gold buckets created and secured")

Reading and writing data

s3_data_operations.py
python
# Upload data to S3 Bronze layer with date partitioning
import boto3
from datetime import datetime

s3 = boto3.client('s3')
today = datetime.now().strftime('%Y/%m/%d')

# Upload raw sales file to Bronze with date-partitioned path
s3.upload_file(
    Filename='sales_raw_20250301.csv',
    Bucket='yourcompany-datalake-bronze',
    Key=f'sales/{today}/sales_raw_20250301.csv'
)

# Reading from S3 in PySpark (AWS Glue or EMR)
from pyspark.sql import SparkSession
spark = SparkSession.builder.getOrCreate()

df = spark.read.option("header", True).csv(
    "s3://yourcompany-datalake-bronze/sales/2025/03/01/"
)

# Write to Silver as Parquet with partitioning
df.write.mode("overwrite").partitionBy("region").parquet(
    "s3://yourcompany-datalake-silver/sales/"
)

print(f"✅ Processed {df.count()} rows to Silver")
🎯 Pro Tip
Always use IAM roles instead of access keys when connecting AWS services to S3. An IAM role attached to your Glue job or EC2 instance grants S3 access without storing any credentials anywhere. This is the AWS equivalent of using Managed Identities on Azure.

S3 Lifecycle Policies — save money on old data

Raw Bronze data is accessed frequently when first ingested, then rarely touched again. S3 Lifecycle Policies automatically move old data to cheaper storage tiers — saving significant costs at scale.

s3_lifecycle_policy.json
json
# S3 Lifecycle Policy — save costs on old data (JSON config)
# Apply to Bronze bucket: move old raw files to cheaper storage tiers
{
  "Rules": [
    {
      "ID": "MoveOldBronzeToInfrequentAccess",
      "Status": "Enabled",
      "Filter": { "Prefix": "sales/" },
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"   
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"        
        }
      ]
    }
  ]
}
S3 Standard
$0.023/GB/month
Frequently accessed (< 30 days)
S3 Standard-IA
$0.0125/GB/month
Infrequently accessed (30–90 days)
S3 Glacier
$0.004/GB/month
Archive — rarely accessed (90+ days)

🎯 Key Takeaways

  • ✓S3 is the AWS data lake — equivalent to ADLS Gen2. Every AWS data pipeline starts with data in S3
  • ✓Use separate buckets for Bronze, Silver, and Gold layers — mirrors the Medallion Architecture exactly
  • ✓Always block public access on all S3 buckets — never make data lake buckets public
  • ✓Use IAM roles (not access keys) for service-to-service authentication — the AWS equivalent of Managed Identities
  • ✓Store data as Parquet (not CSV) in production and partition by date for dramatically faster query performance
  • ✓S3 Lifecycle Policies automatically move old data to cheaper storage tiers — essential for controlling costs at scale
