AWS Glue — Serverless ETL

AWS Glue is a serverless ETL service for transforming data in S3. Write PySpark jobs that run without provisioning any infrastructure, and use the Glue Data Catalog as the metadata store for all your data lake tables.

What AWS Glue does

AWS Glue combines two things that Azure keeps separate: it is the equivalent of both Azure Databricks (for PySpark processing) and Azure Data Factory (for ETL jobs and scheduling) — but as a serverless service. You don't provision a cluster or configure compute. You write a PySpark script, tell Glue how much capacity to use, and it runs.

Glue also includes the Glue Data Catalog — a centralized metadata store that tracks the schema of all your S3 data. Once a table is registered in the Catalog, Amazon Athena and Redshift Spectrum can query it using standard SQL without moving any data.

Glue ETL Jobs

PySpark or Python scripts that transform data in S3. Serverless — Glue provisions compute automatically. You pay per DPU (Data Processing Unit) hour used.

Glue Crawlers

Automatically scan S3 folders, infer the schema, and register the table in the Glue Data Catalog. Run after every pipeline execution to keep metadata current.

Glue Data Catalog

Centralized metadata repository for all your data lake tables. Athena, Redshift Spectrum, and EMR all read from this catalog to discover table schemas.

Glue Workflows

Visual pipeline orchestration inside Glue. Chain crawlers and ETL jobs together with triggers. Simpler but less powerful than AWS Step Functions.

Writing a Glue ETL job

Glue ETL jobs are PySpark scripts with a small Glue-specific wrapper. The core transformation logic is identical to what you write in Databricks — the difference is the initialization boilerplate and the GlueContext instead of SparkContext.

glue_bronze_to_silver.py
python
# AWS Glue PySpark Job — Bronze to Silver transformation
# This runs serverlessly — no cluster management needed

import sys
from awsglue.transforms import *
from awsglue.utils import getResolvedOptions
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from awsglue.job import Job
from pyspark.sql import functions as F

# Initialize Glue context
args = getResolvedOptions(sys.argv, ['JOB_NAME', 'run_date'])
sc = SparkContext()
glueContext = GlueContext(sc)
spark = glueContext.spark_session
job = Job(glueContext)
job.init(args['JOB_NAME'], args)

run_date = args['run_date']  # Passed from Step Functions

# Read from S3 Bronze using Glue DynamicFrame
bronze_dyf = glueContext.create_dynamic_frame.from_options(
    format_options={"withHeader": True, "separator": ","},
    connection_type="s3",
    format="csv",
    connection_options={"paths": [f"s3://yourcompany-datalake-bronze/sales/{run_date}/"]},
)

# Convert to Spark DataFrame for transformations
df = bronze_dyf.toDF()

# Data quality and transformation
df_clean = (df
    .dropDuplicates(["order_id"])
    .filter(F.col("order_id").isNotNull())
    .withColumn("order_date",   F.to_date("order_date", "yyyy-MM-dd"))
    .withColumn("unit_price",   F.col("unit_price").cast("double"))
    .withColumn("quantity",     F.col("quantity").cast("integer"))
    .withColumn("net_amount",   F.col("unit_price") * F.col("quantity"))
    .withColumn("_load_date",   F.lit(run_date))
)

# Write to S3 Silver as Parquet
df_clean.write.mode("append").partitionBy("order_date").parquet(
    "s3://yourcompany-datalake-silver/sales/"
)

print(f"✅ Glue job complete: {df_clean.count()} rows written to Silver")
job.commit()

Glue Crawlers — automatic schema discovery

After your ETL job writes data to S3, run a Glue Crawler to register the table schema in the Data Catalog. This is what makes the data queryable by Athena and Redshift Spectrum without any manual schema definition.

glue_crawler_setup.py
python
# Glue Crawler — automatically discovers schema and registers tables
# After running, the table appears in Glue Data Catalog
# Athena and Redshift Spectrum can then query it using SQL

import boto3

glue = boto3.client('glue', region_name='us-east-1')

# Create a crawler for the Silver layer
glue.create_crawler(
    Name='silver-sales-crawler',
    Role='arn:aws:iam::123456789:role/GlueServiceRole',
    DatabaseName='silver_db',
    Targets={
        'S3Targets': [{'Path': 's3://yourcompany-datalake-silver/sales/'}]
    },
    Schedule='cron(0 6 * * ? *)'  # Run daily at 6am UTC
)

# Start the crawler manually (also runs on schedule)
glue.start_crawler(Name='silver-sales-crawler')
print("✅ Crawler started — will register Silver schema in Glue Catalog")
🎯 Pro Tip
The Glue Data Catalog is shared across Athena, Redshift Spectrum, and EMR — register a table once and query it from any of these services. This is the AWS equivalent of creating an external table in Azure Synapse that points to Delta files in ADLS Gen2.

🎯 Key Takeaways

  • ✓AWS Glue is serverless ETL — write PySpark jobs without provisioning or managing any compute infrastructure
  • ✓Glue ETL jobs use the same PySpark API as Databricks — the core transformation logic is identical
  • ✓The Glue Data Catalog is the central metadata store — register tables once, query from Athena and Redshift
  • ✓Glue Crawlers automatically infer and register schema from S3 data — run them after every pipeline execution
  • ✓DPU (Data Processing Unit) is the billing unit — Glue charges per DPU-hour consumed during job execution
  • ✓For complex orchestration, combine Glue jobs with AWS Step Functions — Glue Workflows are simpler but limited
